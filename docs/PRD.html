1. Executive Summary

TechToons is an interactive, browser-based coloring book that introduces children (ages 4–10) to technology concepts through creative play. Children color tech-themed illustrations — robots, circuit boards, AI brains, rockets, drones, and VR headsets — using intuitive digital tools, building familiarity with STEM imagery while exercising creativity.

Launch Market: Kenya is chosen as the initial market due to its rapidly growing EdTech sector, high mobile internet penetration (~90% of internet access via mobile), a young population (over 40% under age 15), and the Kenyan government's emphasis on digital literacy through the CBC (Competency-Based Curriculum) framework.

Business Goal: Transform the existing prototype into a commercially viable EdTech product targeting Kenyan families and schools, generating recurring revenue through an affordable freemium model priced for the Kenyan market, before expanding across East Africa and beyond.

4. Current Product Capabilities (v0.2 — Enhanced Prototype)

The current build demonstrates core viability plus kid-friendly UX:

Core Features

  • ✅ 6 tech-themed SVG coloring pages (Robot, Circuit Board, AI Brain, Rocket, Drone, VR Headset)
  • ✅ Click-to-fill region painting with inline SVGs
  • ✅ 16-color curated palette + full-spectrum custom picker
  • ✅ 4 brush tools: Paint Bucket (fill), Freehand Brush (draw with finger/stylus), Eraser, Rainbow (random)
  • ✅ 20-level undo stack + full reset
  • ✅ Export to PNG (2× resolution)
  • ✅ Responsive layout (desktop + mobile)
  • ✅ Clean design system (Fredoka + Nunito, teal/coral palette, 8px grid)
  • ✅ Deployed on Vercel

Kid-Friendly UX NEW in v0.2

  • ✅ Labeled Toolbar — Visible text labels on every toolbar control ("Paint", "Erase", "Rainbow", "Undo", "Start Over", "Save") with section headings ("Colors", "Tools", "Actions")
  • ✅ Live Region Label Indicator — Dynamic indicator above the canvas ("🎨 You're coloring: Head") that updates on hover/click, so kids know exactly what part they're coloring
  • ✅ Region Label Maps — 100+ human-readable region names across all 6 pages (e.g., "Antenna Ball", "Processor (CPU)", "Left Thinking Area", "Nose Cone", "Camera Lens", "Left Controller")
  • ✅ Educational Descriptions — Each coloring page shows an age-appropriate description explaining the tech concept (e.g., "A robot has a head, arms, and legs — just like you! Can you color each part?")
  • ✅ Gallery Descriptions — Home page cards show educational blurbs so kids understand what they'll learn before clicking
  • ✅ Guided Onboarding Prompt — Default state shows "👆 Tap a part to start coloring!" to guide new users
  • ✅ Color Name Labels — Selected color displays its name ("Red", "Teal", "Violet", etc.) next to the color preview for learning

  • react-colorful (color picker)
  • react-router-dom v7 (client-side routing)

5. Feature Requirements

5.1 Phase 1 — Foundation (Months 1–3)

Goal: Transform prototype into a launchable product

FR-1: User Accounts & Authentication P0 — Must Have

Parents create accounts; children have sub-profiles under a parent account.

  • Email/password + Google OAuth sign-up
  • Phone number sign-up (common in Kenya)
  • Up to 4 child profiles per parent account
  • Each child profile has name, avatar, age
  • Child profiles have independent saved progress
  • Kenya Data Protection Act (DPA 2019) compliant data collection

Acceptance Criteria: Parent can create account, add child profiles, and switch between them

FR-2: Cloud Save & Progress Persistence P0 — Must Have

Coloring progress is automatically saved and synced across devices.

  • Auto-save every 30 seconds + on page leave
  • Region color state stored per user per page
  • Gallery of completed works per child profile
  • Offline support with sync-on-reconnect

Acceptance Criteria: User can close tab, return later, and resume coloring from where they left off

FR-3: Expanded Content Library P0 — Must Have

Grow from 6 pages to 20+ across themed collections.

  • New collections: Coding Creatures, Internet World, Gaming Galaxy, Green Tech
  • 5 pages per collection (20+ total at launch)
  • Difficulty levels: Easy (large regions), Medium, Challenging (detailed regions)
  • 3 free pages + rest behind paywall

Acceptance Criteria: Content library shows collections; free users can access 3 pages, paid access all

FR-4: Freemium Subscription Model P0 — Must Have

Monetize through tiered subscription plans priced for the Kenyan market.

  • Free Tier: 3 coloring pages, basic tools, no save/export
  • TechToons Plus (KES 250/mo ~ $1.99): All pages, all tools, cloud save, PNG export, no ads
  • TechToons Classroom (KES 500/mo ~ $3.99): Plus features + class management, up to 30 students, progress reports
  • M-Pesa integration via Daraja API (primary payment in Kenya)
  • Stripe for card payments
  • Google Play in-app purchase for mobile
  • 7-day free trial for Plus
  • Annual discount (2 months free)

Acceptance Criteria: User can subscribe via M-Pesa or card, access gated content, and manage subscription

5.2 Phase 2 — Engagement & Learning (Months 4–6)

Goal: Increase retention and add educational value

FR-5: Learning Sidebars ("Did You Know?") P1 — Should Have

Each coloring page includes age-appropriate tech facts and mini-lessons.

  • Collapsible sidebar with 3–5 facts per page
  • Read-aloud audio for younger children
  • "Quiz Me" button with simple multiple-choice
  • Facts reviewed by education advisor

Acceptance Criteria: Sidebar appears on coloring page; read-aloud plays; quiz scores display

FR-6: Achievements & Rewards System P1 — Should Have

Gamified reward system to encourage exploration and completion.

  • Badges: "First Robot!", "Color Explorer" (5 pages), "Tech Master" (all pages)
  • Streak tracking (consecutive days)
  • Virtual sticker rewards collected in a sticker book
  • Celebratory animations on milestone completion

Acceptance Criteria: Badges appear in profile; streak counter visible; stickers collectible

FR-7: Parent Dashboard P1 — Should Have

Parents can monitor child engagement and learning progress.

  • Time spent per session and per page
  • Pages completed and collections progress
  • Quiz scores and facts learned
  • Weekly email summary (opt-in)
  • Screen time limits (optional)

Acceptance Criteria: Dashboard displays charts and stats; email summary sends on schedule

FR-8: Sharing & Social Gallery P2 — Nice to Have

Kids can share completed artwork in a moderated community gallery.

  • "Share to Gallery" button on completed works
  • Moderated (AI + human review) before publishing
  • Kids can "star" others' artwork (no comments for safety)
  • Share link to family via email/message
  • No personally identifiable info displayed

Acceptance Criteria: Artwork appears in gallery after moderation; star counts visible

FR-9: Brush Boundary Guidance ("Paint Inside the Lines") P1 — Should Have

When using the freehand brush tool, detect if the child draws outside the coloring regions and display a friendly, encouraging nudge.

  • Real-time boundary detection using canvas hit-testing against SVG region paths
  • Friendly animated messages: "Oops! Try coloring inside the lines 🎯" or "Almost! Stay inside the shape 😊"
  • Gentle toast notification (non-blocking), auto-dismisses after 3 seconds
  • Haptic feedback on mobile devices when painting outside
  • Optional "Training Mode" with highlighted region borders for younger children
  • Configurable sensitivity (tolerant for ages 4–5, precise for ages 8–10)
  • No penalty — purely guidance, not punishment

Acceptance Criteria: Child paints outside a region boundary → friendly message appears within 500ms; message disappears after 3s; no state loss

FR-10: AI Coloring Feedback & Scoring P1 — Should Have

AI-powered feedback system that scores completed artwork, provides encouragement, and suggests improvements.

  • Scoring engine: Evaluate coloring based on coverage (% of regions filled), neatness (brush strokes within boundaries), creativity (color variety), and completeness
  • Score display: Fun, gamified score (e.g., ⭐⭐⭐ out of 5 stars, or "Amazing Artist!" / "Color Explorer" / "Keep Going!")
  • AI suggestions: Gentle, age-appropriate correction prompts (e.g., "The rocket's nose cone is still white — try coloring it!" or "Great use of colors! Try using more than 3 different colors next time")
  • Celebration animations: Confetti / fireworks on high scores
  • Progress over time: Track improvement across sessions; show "You're getting better!" trends in the parent dashboard
  • No negative feedback: All messaging is positive and encouraging — "almost there!" instead of "wrong"
  • CBC alignment: Scores map to CBC competency levels for teacher reports

Acceptance Criteria: Child completes a page → AI score appears with stars and personalized feedback; suggestions are age-appropriate; scores persist in profile

5.3 Phase 3 — Scale & Platform (Months 7–12)

Goal: Expand platform reach and revenue channels

FR-11: Mobile Apps (iOS & Android) P1 — Should Have

Native mobile apps wrapping the web experience with touch-optimized controls.

  • React Native or Capacitor wrapper
  • Touch gestures: tap-to-fill, pinch-to-zoom, swipe navigation
  • Offline mode with local storage
  • Push notifications for streaks and new content
  • In-app purchases for subscriptions

Acceptance Criteria: App published on App Store and Google Play; feature parity with web

FR-12: Educator / Classroom Tools P1 — Should Have

Tools for teachers to use TechToons in classroom settings.

  • Class creation with join codes
  • Assign specific pages as "homework"
  • View class-wide progress and individual reports
  • Curriculum alignment tags (Kenya CBC — Digital Literacy strand)
  • Bulk student account creation via CSV

Acceptance Criteria: Teacher creates class, assigns page, views class progress report

FR-13: Printable Worksheets P2 — Nice to Have

Generate print-ready PDF worksheets from any coloring page.

  • "Print" button exports SVG as black-and-white PDF
  • Includes page title and fun facts on the worksheet
  • A4 and Letter size options
  • Premium feature (Plus subscribers)

Acceptance Criteria: PDF downloads with clean line art and facts

FR-14: API & Partner Integrations P2 — Nice to Have

Open API for embedding TechToons in partner platforms.

  • Embeddable widget (iframe) for partner sites
  • API key authentication
  • Custom branding options for partners
  • Revenue share tracking

Acceptance Criteria: Partner embeds widget; usage tracked in admin panel

Pricing Justification

  • Priced for Kenyan families: KES 250/mo is below the cost of a single meal at KFC (KES ~600)
  • Global competitors (Toca Boca at KES 650/mo, ABCmouse at KES 1,700/mo) are unaffordable for most Kenyan families
  • M-Pesa is the primary payment method — frictionless for 90%+ of Kenyan adults
  • Classroom pricing allows schools to fund from per-learner capitation grants (KES 1,420/pupil/yr)
